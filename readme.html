A) Feature Summary - top

The Drag N' Drop form builder was first written in the .Net framework and then ported over to PHP after popular demand. It provides a lightweight and user-friendly tool for creating almost infinite configurations for end-user forms. Using the simplicity of drag-and-drop and user experience patterns available in most integrated development environments, you can customize forms in several ways. Below are some of the features available:

Feature List

  1. Choose from 17 different draggable fields to create your form
  2. Combination fields (fields consisting of one or more fields combined eg. Address)
  3. Customize required field validation
  4. Add tooltips as a help-resource
  5. Easy to populate drop-down-lists (simply type comma separated list)
  6. Real-time updating
  7. Combination fields - made up of several fields in one eg. Address Field
  8. Autosave - background save process triggers intermittently as you create your form
  9. Export registrant list to excel spreadsheet

C) Technical Summary & Pre-requisites - top

This version of the Drag N' Drop form-builder is built the PHP/MySQL stack. The project comes with scripts that you can use to create the necessary tables in a MySQL database. To upgrade from an older version see the upgrade section below.

To successfully integrate the form-builder into your application with optimal results you will need:
  1. PHP Version 5.4.23
  2. MySQL Community Server (GPL) - version: 5.5.34*
  3. phpMyAdmin (or your preferred user interface for MySql)
  4. JQuery 1.7.1

E) How to integrate form-builder into your application - top

You can, install the form-builder into your existing PHP project by running the SQL scripts that create the application's tables against an existing database instance and copying over the files that make up the rest of the application

Install the database tables

From your instance of phpMyAdmin open the SQL panel, locate the file:

  • ...\FormBuilder\db\1-create-tables.sql*
See the upgrade section below on how to upgrade from an older version. Copy the contents to the text area of the SQL panel and run the queries. After this operation, the following tables should be created.

Copy Over Application Files

You must copy all the folders in the application folder into your project. If you have existing folders, you will have to merge the contents of those folders with the contents of the form builder's folders. If you are an experienced developer you may change the location of files in the project, but note that you will have to look for and change all references to those files within the application.

Updating configurations

Configurations for the application can be found in the file:

How to manage Themes for your forms

The form-builder allows you to customize the look-and-feel of the forms you create. All styles that control the look and feel of forms are stored in the path:

\Content\css\form-builder\themes

To create a new theme, simply create a new directory within that folder and place a "style.css" document within it. You can use an existing theme's style.css file to guide you in creating your themes.

How to update templates

Templates are preset forms that you can use as a starting-step to create your own forms. They save you some work by having the fields needed for certain forms already added. Once installed, you can create forms from templates by selecting the template from the dropdownlist on the form listings page and clicking "Add Form".

To install templates, you must have template files (json files in the "template-files"" folder of your root directory) installed. You should see an "Update Templates" button on the form listing page. If not, check the setting "SHOW_UPDATE_TEMPLATES_BUTTON" and make sure it's set to true. Once visible, you can click on the "Update Templates" button and all available templates will be read from the template-files folder and imported into your database.

Behind the scenes, templates work just like forms with a special setting to mark them as templates. You can manage your templates like you do regular forms by setting the setting "SHOW_TEMPLATES_IN_LIST" to true. You can also turn some of your own forms into templates with this setting as it reveals a checkbox that allows you to check/uncheck which forms can be made into templates.

Upgrade

To upgrade from a previous version, locate the db folder from the root of the application and navigate to the folder with the versions you are trying to upgrade from and to. The folder name will be in the format

"upgrade-from-version-{base-version}-to-{upgrade-version}"

So for example, to upgrade from version 1 to 2, you will need to locate the "upgrade-from-version-1-to-2" folder. From the upgrade folder, locate a sql file of the same name, and run it against your base version database in MySql

FAQ

How do I create a form?
To create a form, go to the index page of your form builder, and click on "Add Form"

How do I rename a form?
To rename a form, select a form from the index page. While in the form's edit mode, you can click on the form's title label and edit the form's name in place or in the edit panel, edit the "Title" property and hit enter

How do I edit a drop-down list field?
Drop-down-lists are populated by comma-delimited lists in the edit pane. The first item in the list is usually selected by default unless you specify the "Selected Option" field

How do I delete a form?
You can only delete a form if there are no form submissions. For a form with no submissions, go to the index page of your form builder instance and click on delete to remove the form.

How do I fill out a form?
To fill out a form, visit the index page of your form builder instance and click on the "View/Fill Form" button. You can then fill out and submit the form.

How do I export registration data to excel?
To export form data to excel, you must have one or more form submissions. From your form's index page, click on "View Submissions" button, then on the submissions page, click on the "Export All to Excel" button.

How do I manage themes for my forms?
To manage themes, create a new directory (eg. "Awesome New Theme") in the "/Content/css/form-builder/themes/" directory. Then create a new css file in that directory called "styles.css". You can then write css to customize the look and feel of your forms. You can style each individual form differently.
Tip: Use an existing theme's style.css file as a guide to help you create your own

How do I manage notification email settings?
You can manage where email notifications are sent to after a user submits an entry to your form. You can determine which email you want form submissions sent to, on a form-by-form basis. To edit the email address entry-notifications are sent to, click on the title of the form to select it, then in the edit panel, enter an email address into the notification email box. This will be the address that submissions notifications are sent to.

To edit the smtp settings the application uses to send email, open the /lib/forms-settings.php and edit the smtp config settings to use your email server's details.

How do I manage where the file-upload control stores uploaded files?
To edit the path that files uploaded using the file-upload control of your forms are stored, first determine where you would like to store uploaded files. You can choose to store uploads either on the Amazon S3 cloud or in your local directories.

To store files in your local directories, open the lib/forms-settings.php file and set the "USE_CLOUD_STORAGE" setting to "false". Then set the "FILE_SAVE_PATH" setting key to any directory you would prefer to store file uploads in (note: you must use a relative path).

To store files on the Amazon S3 cloud set the "USE_CLOUD_STORAGE" application setting to "true" and update the configuration settings for authenticating against the Amazon S3 cloud.
